History API
Routing with HTML5 History API

window.history object provides several methods for manipulating the history of the browser. Window.history.back() can be used for moving a step back or window.history.forward() to do the opposite. And window.history.pushState method can create a new entry in the history of the browser.

Going back in the history, makes a popstate event be fired and the previous state avaliable.

Hash-based
Routing with hash-based strategy

Hash-based routing makes use of the anchor part of the URL to simulate different content. In the url http://example.com/#/creatures/list, the part right after hash simbol #/creatures/list, is never sent to the server.

The routing is possible because changes in the hash don't trigger page reload, however triggers hashchange window event, an opportunity to render specific content.
